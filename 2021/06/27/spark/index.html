累加器

累加器的作用是将driven与executor连接起来，并且累加executor返回的值

一般情况下，累加器会放置在行动算子进行操作

为什么要用Yarn

Spark 作为计算框架，Yarn能作为资源调度框架，调配自源。利用ResourceManager和NodeManager进行自源调配。 Yarn有两种模式， Client,Cluster， 区别是driver的运行节点位置。

RDD

RDD是最小可计算单元，可继续分为Task给Executer执行。 RDD类似于层层封装，只有使用到Collect 才会正真执行，RDD不能做Buffer临时保存。RDD比如是Text File， Map等，这些最小的计算单元，RDD的Partition可分成Task然后传给Executor

Map 与mapPartition

Map 一个一个处理，速度较慢，map传入多少条数据，返回多少条数据

lines.map(_*2)

mapPartition 在内存中处理，将分区进行批处理，但是有可能造成内存溢出，可得到分区的最大值，返回与传入的数据条数可以不一致

lines.mapPartition(iter=>iter.map(_*2))

Glom

分区变成一个List

groupBy

得到的时一个（word, iter :List[string,int]）的集合。打乱了数据，同一组的一定在同一分区,groupBy(word=>word)

结果是（word，Iter(word,Int)）

groupByKey

得到的时一个（word, iter :List[int]）的集合。打乱了数据，同一组的一定在同一分区,groupByKey()

结果是（word，Iter(Int)）

存在shuffle操作，中间存在落盘的操作，为了防止内存溢出，IO会导致性能降低，shuffle一定有落盘

Filter

留下满足要求的，数据保留在原来的分区，所以可能会导致数据倾斜

Sample

抽样，可以用于判断数据倾斜的原因

Coalesce

扩大或者缩减分区，减少数据调度成本，比如每个分区数据都很少，可以合并成一个，减少调度成本。不会打乱数据重新组合，所以有可能导致数据倾斜，但是可以配合shuffle来重新组合，避免数据倾斜。shuffle默认为false， 扩大分区必须配合shuffle，否则没有意义，但是主要用于缩减分区

Repartition

主要用于扩大分区，shuffle默认为false

sortBy

利用规则排序，默认为升序

Union, Intersection, Subtract, Zip

Union会保留两个相同的并集，即会出现重复

Subtract以左边的为主，减去并集

Union,Intersection,Subtract, 需要类型相同

Zip将同位置的zip成一个tuple,分区数量需要相同，数据数量相同

PartitonBy

(key,value)的重新分区,分区器有hashpartition和rangepartition，但是也可以自己写

rdd.partitionBy(new HashPartitioner(2))

foldByKey

分区内与分区间的计算规则相同，且可定义初值

CombineByKey

第一个参数：转换第一个值的类型，比如， int => （int，int）

第二个参数：分区内计算

第三个参数，分区间计算

所有的ByKey类型，底层其实都是combineByKey

血缘关系

rdd.toDebugString

窄依赖，宽依赖

窄依赖：one to one依赖，不需要划分阶段，因为不需要等待其他分区

宽依赖：shuffle 依赖，需要划分阶段

阶段数量=shuffle数量(ShuffleMap Stage)+1（Result Stage）

job是每个Action算子就是一个job

Task是每个阶段的最后一个RDD的分区个数就是Task数，一般跟分区数一样(ShuffleMap Task, Result Task)

Cache, persist(storageLevel.DISK_ONLY), checkpoint

Cache: 速度快，存在内存中，但是可能会溢出，数据可能不安全，临时文件,会在血缘关系中添加关系，一旦发生问题，从头重读恢复

DISK_ONLY:涉及IO,速度慢，但是数据安全，临时安全

checkpoint:数据长久保存在磁盘中，涉及IO,速度慢，但是数据安全，会独立执行作业，为了提高效率，一般与cache联合使用， rdd.cache()+rdd.checkpoint()。切断血缘关系，重新建立血缘，等同于改变数据源

累加器

聚合各个executor的结果，否则每个分区都是独立的，无法获得最终的结果，其实就是把executor的结果聚合到driver端。为只写功能，即executor互相不能看到对方

广播

分布式中的只读，所以的task公用，减少数据重复
